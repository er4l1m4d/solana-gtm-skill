## The Framework: 4-Step Competitive Analysis

### Step 1: Map the Competitive Landscape

**Guiding Questions:**
1. Who are your direct competitors? (Same product, same market)
2. Who are your indirect competitors? (Different product, same problem)
3. Who are your potential competitors? (Could enter your market)
4. What categories do they fall into?

**Competitive Landscape Map:**

```
                    Direct Competitors
                          │
                          │
    Indirect Competitors ──┼── Potential Competitors
                          │
                          │
                    Your Position
```

**Solana Ecosystem Categories:**
| Category | Examples | Overlap With |
| :--- | :--- | :--- |
| DEX / Perps | Jupiter, Drift, GMTrade, Raydium | Trading, liquidity |
| Lending | Kamino, Solend, Port Finance | Borrowing, leverage |
| NFT / Tokens | Metaplex, Magic Eden, Tensor | Digital assets |
| Infrastructure | Helius, QuickNode, Triton | Dev tools, RPC |
| Wallets | Phantom, Solflare, Backpack | User onboarding |
| Stablecoins | PYUSD, USDC, UXD | Payments, stability |

### Step 2: Build the Competitive Matrix

**Guiding Questions:**
1. What are the top 3-5 competitors in your category?
2. What do they do well? (Strengths)
3. Where do they fall short? (Weaknesses)
4. How do you compare on key dimensions?

**Competitive Matrix Template:**

| Feature | You | Competitor A | Competitor B | Competitor C |
| :--- | :--- | :--- | :--- | :--- |
| **Core Product** | [What you do] | [What they do] | [What they do] | [What they do] |
| **Fees** | [Your fee] | [Their fee] | [Their fee] | [Their fee] |
| **Speed** | [Your speed] | [Their speed] | [Their speed] | [Their speed] |
| **Assets** | [Your assets] | [Their assets] | [Their assets] | [Their assets] |
| **UX** | [Your UX] | [Their UX] | [Their UX] | [Their UX] |
| **TVL** | [Your TVL] | [Their TVL] | [Their TVL] | [Their TVL] |
| **Users** | [Your users] | [Their users] | [Their users] | [Their users] |
| **Unique** | [Your advantage] | [Their advantage] | [Their advantage] | [Their advantage] |

**Example (GMTrade):**

| Feature | GMTrade | Drift | Jupiter Perp | GMX (Ethereum) |
| :--- | :--- | :--- | :--- | :--- |
| **Core Product** | Multi-asset perps | Order book perps | Aggregated perps | Pooled perps |
| **Fees** | 0.5 bps | 2-5 bps | 1-3 bps | 10-30 bps |
| **Speed** | 400ms | 400ms | 400ms | 12s |
| **Assets** | 90+ (crypto, FX, commodities) | 30+ (crypto only) | 20+ (crypto only) | 20+ (crypto only) |
| **UX** | CEX-like | Advanced | Simple | Advanced |
| **TVL** | $10M+ | $100M+ | $50M+ | $500M+ |
| **Unique** | RWA + low fees | Order book depth | Aggregation | Ethereum liquidity |

### Step 3: SWOT Analysis

**Guiding Questions:**
1. What are your strengths? (Internal advantages)
2. What are your weaknesses? (Internal disadvantages)
3. What are the opportunities? (External advantages)
4. What are the threats? (External disadvantages)

**SWOT Template:**

| | Positive | Negative |
| :--- | :--- | :--- |
| **Internal** | **Strengths** | **Weaknesses** |
| | - [Strength 1] | - [Weakness 1] |
| | - [Strength 2] | - [Weakness 2] |
| | - [Strength 3] | - [Weakness 3] |
| **External** | **Opportunities** | **Threats** |
| | - [Opportunity 1] | - [Threat 1] |
| | - [Opportunity 2] | - [Threat 2] |
| | - [Opportunity 3] | - [Threat 3] |

**Example (GMTrade SWOT):**

| | Positive | Negative |
| :--- | :--- | :--- |
| **Internal** | **Strengths** | **Weaknesses** |
| | - 0.5bps fees (5-9x cheaper) | - New project (low TVL) |
| | - 90+ markets including RWA | - Limited track record |
| | - Solana speed + low costs | - Smaller team |
| **External** | **Opportunities** | **Threats** |
| | - RWA market exploding | - Drift improving fees |
| | - Solana ecosystem growth | - Jupiter adding perps |
| | - Mobile trading trend | - Regulatory uncertainty |

### Step 4: Define Your Positioning

**Guiding Questions:**
1. What's your unique value proposition?
2. What's your competitive advantage?
3. How do you want to be perceived?
4. What's your positioning statement?

**Positioning Statement Template:**

```
For [TARGET AUDIENCE]
who [HAVE THIS PROBLEM],
[PROJECT NAME] is a [CATEGORY]
that [KEY BENEFIT].
Unlike [COMPETITORS],
we [UNIQUE DIFFERENTIATOR].
```

**Example Positioning Statements:**

| Project | Positioning Statement |
| :--- | :--- |
| GMTrade | "For retail traders who want CEX-like experience on-chain, GMTrade is a multi-asset perpetuals platform that offers 0.5bps fees and 90+ markets including RWA. Unlike Drift or Jupiter Perp, we support real-world assets like gold, oil, and FX." |
| Kamino | "For DeFi users who want simple leverage, Kamino is a lending and leverage protocol that offers one-click leverage on Solana. Unlike Solend, we provide automated leverage strategies." |
| Pyth | "For developers who need reliable oracle data, Pyth is a decentralized oracle network that offers sub-second price feeds for 100+ assets. Unlike Chainlink, we pull data directly from first-party publishers." |

## Positioning Strategies

### Strategy 1: Cost Leadership
- **Position**: "The cheapest option"
- **Example**: GMTrade (0.5bps fees)
- **Risk**: Race to the bottom, margin pressure

### Strategy 2: Differentiation
- **Position**: "The only one that does X"
- **Example**: GMTrade (RWA support)
- **Risk**: Competitors copy your features

### Strategy 3: Niche Focus
- **Position**: "The best for [specific use case]"
- **Example**: Kamino (leverage on Solana)
- **Risk**: Market too small, competitors expand

### Strategy 4: Ecosystem Integration
- **Position**: "The most integrated with [ecosystem]"
- **Example**: Jupiter (aggregated across Solana DEXs)
- **Risk**: Ecosystem dependency, partner competition

### Strategy 5: User Experience
- **Position**: "The easiest to use"
- **Example**: Phantom (simplest wallet)
- **Risk**: UX is hard to defend, competitors improve

## How to Win Against Competitors

### Against Larger Competitors
1. **Move Faster**: Ship features they can't (smaller team, less bureaucracy)
2. **Be More Agile**: Pivot quickly based on user feedback
3. **Focus on Niche**: Own a specific use case they ignore
4. **Build Community**: Create passionate users they can't match

### Against Similar-Sized Competitors
1. **Differentiate**: Find a unique angle they don't have
2. **Out-Execute**: Better product, better UX, better support
3. **Partner Smart**: Secure exclusive partnerships
4. **Build Moat**: Create switching costs (data, network effects)

### Against Newer Competitors
1. **Leverage Brand**: Use your established reputation
2. **Highlight Experience**: Show your track record
3. **Improve Continuously**: Don't rest on your laurels
4. **Acquire or Partner**: Consider acquiring or partnering
